### âŒ Step 2: Reject the First 3 Obvious Ones (Forced Discard)

**This is the core of MOLD BREAKER.** Explicitly discard the 3 most typical candidates.

For each discarded candidate, state the rejection reason in one line:

| Rejection Reason | Example |
|-----------------|---------|

**Why 3?** Research on ideation consistently shows that the first ideas in any generation process cluster around high-probability solutions. By discarding 3 of 5, you eliminate the probability peak and force attention to the tails.

### âœ… Step 3: Select Top 2 + Hybrid (Keep the Edge)

**Hybrid check:** Can elements from the two selected candidates be recombined into a third option? If yes, propose it. If no, move on.

## 4. Typicality Assessment Scale

Use this scale to evaluate any output's typicality:

| Level | Definition | AI Generation Probability | Example (SaaS pricing redesign) |
|-------|-----------|--------------------------|--------------------------------|
| **High** | First association, template response | >60% | "3-tier table with Free/Pro/Enterprise" |
| **Medium** | Slight twist, but still familiar | 30-60% | "Usage-based pricing with calculator widget" |
| **Low** | Unexpected but coherent | 10-30% | "Price revealed only after value demonstration" |
| **Lowest** | Structurally novel, challenges assumptions | <10% | "Customer sets own price; margin shared transparently" |

**Mandate:** Final output must target **Low** or **Lowest**. If everything is Medium or above, run the protocol again with different Killer Constraint.
