# Control Experiments: Protocol vs Standard Methods

**目的**: 同一の入力Fragmentsに対して、GHOSTY/PRECOGプロトコルを使った場合と、標準的な手法（ブレインストーミング/SWOT）を使った場合の出力を比較する。

---

## Experiment 1: Music Production — 同一入力、異なるプロセス

### 入力（共通）
- K-popダンスグループの新曲コンセプト
- テーマ: 「好きを隠す」
- 参考ジャンル: Jersey Club / UK Garage
- メンバー5人の個別キャラクター設定

### Control（標準ブレインストーミング）

**プロセス**: テーマから連想 → ジャンル選択 → 歌詞作成 → アレンジ

**結果**:
- コンセプト: 「秘密の恋心をダンスで表現するポップチューン」
- Key: G minor（「エモい」の定番選択）
- BPM: 122（Jersey Club標準帯）
- 構成: Verse→Chorus→Verse→Chorus→Bridge→Chorus（定番構成）
- 転調: Final Chorusで半音上げ（「盛り上がるから」）
- 歌詞設計: 意味ベース（「気持ちを隠す→でもバレちゃう→笑顔」の直線構造）
- ボーカルディレクション: なし

**出力品質**: 機能的に正しい。商業的に成立する。だが**既存のK-pop楽曲と構造的に区別不能**。

### Treatment（GHOSTY COLLIDER経由）

**プロセス**: Fragment Harvest → Ghost Extraction → Collision Matrix → Vision → Reality Bridge

**結果**:
- コンセプト: 「鎧をまとった告白」— 隠す行為自体がM的快楽
- Key: **F# minor**（「孤独、内向的強度」— Key特性表から導出）
- BPM: **128**（K-pop dance最適帯 + UK Garage親和性から導出）
- 構成: 同じだが**各セクションにVA座標を定義**。エネルギー曲線を設計
- 転調の意味: **「覚醒」ではなく「白旗」** — Ghost「隠す快楽の終焉」から導出
- 歌詞設計: **音響ベース** — 「좋아해」の母音ㅗ→ㅏ→ㅐが口の開き方=感情の開放度
- ボーカルディレクション: 全9セクション定義。Final Chorusで「声が割れてもいい」= 崩壊の入力

### 差分分析

| 評価軸 | Control | Treatment | 差分の原因 |
|--------|---------|-----------|-----------|
| コンセプトの抽象度 | 表面的（隠す→バレる） | 深層的（隠す行為の快楽性） | Ghost Extraction |
| Key選択の根拠 | 慣習（「エモいから」） | 理論的（Key特性表 + テーマ） | Vision→パラメータ変換 |
| 転調の意味 | 定番（盛り上がり） | 再定義（白旗/崩壊） | Collision（Ghost1×5） |
| 歌詞の設計単位 | 意味（言葉の内容） | 音響（母音の物理特性） | Ghost「母語でしか出せない感情」 |
| 全体の独自性 | 既存楽曲と区別困難 | **構造的に新しいアプローチ** | Collision Matrixの交差 |

## Experiment 2: Filmmaking — 同一入力、異なるプロセス

### 入力（共通）
- 5人のキャラクターが登場する60秒ショートフィルム
- 縦動画フォーマット（9:16）
- 目標: SNSでの視聴停止率の最大化

### Control（標準企画ブレスト）

**プロセス**: ターゲット分析 → ベンチマーク → コンセプト → 構成

**結果**:
- コンセプト: 「5人の個性が映える華やかなダンス映像」
- 構成: 全員のダンスショット → ソロカット → フォーメーション → ロゴ
- フック: 冒頭にインパクトのあるダンスムーブ
- 照明: 多灯、カラフル
- 音楽: 楽曲BGMあり
- カメラ: 動きのある手持ち + クレーンショット

**出力品質**: 視覚的に華やか。商業的に正しい。だが**K-popのMVと構造的に同一**。

### Treatment（GHOSTY COLLIDER経由）

**結果**:
- コンセプト: 「動かない映像が不穏に心を掴む」
- 構成: **1人1カット×12秒の5カット。各カットに独自のテンション設計**
- フック: **静止で不穏な直感を与える**（動きではなく質感で殴る）
- 照明: **単一光源、影が主役**（Ghost「崩壊の途中の美」→退色=彩度-30%）
- 音楽: **無音。環境音のみ**（Ghost「注意の暴力」→聴覚ではなく視覚だけで拘束）
- カメラ: **完全固定**（Ghost「凍結された崩壊」→フレームが額縁=崩壊を強調）

### 差分分析

| 評価軸 | Control | Treatment | 差分の原因 |
|--------|---------|-----------|-----------|
| フック設計 | 動きで掴む（定番） | 静止で掴む（反転） | Ghost「注意の暴力」 |
| 照明 | 華やか（安全策） | 単一光源（不穏） | Ghost「崩壊の途中の美」 |
| 音響 | BGMあり（安全策） | 無音（冒険） | Collision「動かないことが最も動的」 |
| メンバー活用 | 全員同時（横並び） | 1人1カット（個別深掘り） | Ghost Extractionで個別の深層構造を発見 |
| 独自性 | K-pop MV標準に準拠 | **K-pop映像の文法を反転** | Collision Matrixの交差 |

## Experiment 3: Trend Forecasting — 同一入力、異なるプロセス

### 入力（共通）
- AI Agent市場の2026-2028年予測
- 公開情報（OpenAI/Anthropic/Google/Microsoft発表、EU AI Act等）

### Control（標準SWOT分析）

**プロセス**: 強み/弱み/機会/脅威を列挙 → 戦略方向性を提案

**結果**:
| S (強み) | W (弱み) |
|---------|---------|
| Big4が参入済み→市場は確実 | ハルシネーション問題 |
| 開発ツール充実 | 人材不足 |

| O (機会) | T (脅威) |
|---------|---------|
| 業務自動化需要 | 規制強化 |
| MCP標準化 | プライバシー懸念 |

**戦略提案**: 「エージェント市場に参入し、差別化されたソリューションを提供すべき」

**出力品質**: 網羅的だが**時間軸がない**。「いつ」「何をトリガーに」が欠落。静的スナップショット。

### Treatment（PRECOG PROTOCOL経由）

**結果**:
- **Signal Map**: 8信号を Strong/Emerging/Weak に分類 + Confidence判定
- **Convergence**: 「エージェントの"ブラウザ元年"」仮説 — 複数シグナルの交差点を構造化
- **Contrarian**: 3つの逆張り仮説 + 確率付き（25-35%, 30-40%, 50-60%）
- **Timing Grid**: 4軸（市場/競争/技術/規制）の多次元マトリクス → 「Early Go」判定
- **Action Window**: Now/Soon/Watch/Kill の4区分 + **各アクションにトリガー条件**

### 差分分析

| 評価軸 | Control (SWOT) | Treatment (PRECOG) | 差分の原因 |
|--------|---------------|-------------------|-----------|
| 時間軸 | なし（静的） | あり（Now/Soon/Watch/Kill） | Timing Grid構造 |
| 逆張り | なし | 3仮説 + 確率推定 | Contrarian Viewステップ |
| 信号の分類 | S/W/O/Tの4象限 | Strong/Emerging/Weak + 収束分析 | Signal Map + Convergence |
| アクション精度 | 「参入すべき」（曖昧） | 具体的アクション + トリガー条件 | Action Window構造 |
| 意思決定への寄与 | 「状況はわかった」で終わる | **「今やること」が明確に出る** | プロトコル全体の設計思想 |

| 比較軸 | Standard Methods | Protocol-Driven |

| 出力の正確性 | ✅ 正確 | ✅ 正確 |
| 出力の網羅性 | ✅ 網羅的 | 🟡 焦点を絞る |
| 出力の独自性 | ❌ 既存パターンに収束 | ✅ **構造的に新しい視点** |
| 出力の実行可能性 | 🟡 「何をすべきか」は曖昧 | ✅ **具体パラメータに落ちる** |
| プロセスの再現性 | ❌ 参加者依存 | ✅ **手順が形式化されている** |
| 時間軸の有無 | ❌ 静的 | ✅ 動的（タイミング判定） |

| 超越方向数 | 1-2（増幅のみ） | **4**（増幅+深化+反転+破壊） | +100-300% |
